## 제1부: 고수는 이렇게 씁니다

**부제:** "다섯 사람의 하루"

> 1부의 전략: 노트북LM이 무엇인지 설명하기 전에, 이미 쓰고 있는 사람들의 모습을 먼저 보여준다. "저게 뭐지?" 하는 호기심을 만드는 것이 목적이다. 네 사람의 스토리는 프롤로그와 에필로그에서 다시 등장하여 전체 서사를 완성한다.

### 제1장: 200페이지 보고서를 5분 만에 읽는 팀장 (4,000~5,000자)

**하위 섹션:**
- 월요일 아침, 200페이지 보고서가 메일함에 도착했다
- "전체 요약해줘"가 아니라 "이 보고서에서 우리 팀에 영향 있는 부분만 골라줘"
- 팀장이 사용한 핵심 기법: 소스 업로드 → 맞춤 질문 → 노트 저장
- 이 팀장이 노트북LM을 쓰기 전과 후의 하루 비교
- [그림: Before/After 비교, 팀장의 하루 타임라인]

**핵심 정리:** 노트북LM은 모든 것을 요약하는 도구가 아니다. 내가 궁금한 것만 골라서 답해주는 도구다.

### 제2장: 임원 보고 전날 밤을 바꾼 기획자 (4,000~5,000자)

**하위 섹션:**
- 임원 보고 D-1, 세 개 부서의 보고서를 합쳐야 한다
- 소스 세 개를 올리고, "이 세 보고서의 공통점과 차이점을 표로 정리해줘"
- 기획자가 사용한 핵심 기법: 교차 비교 분석 → 슬라이드 자동 생성
- 밤샘 기획이 오후 퇴근으로 바뀐 과정
- [그림: 세 개 보고서가 하나의 비교표로 합쳐지는 플로우차트]

**핵심 정리:** 여러 문서를 비교 분석하는 것이야말로 노트북LM이 가장 잘하는 일이다.

### 제3장: 회의 끝나고 5분 만에 액션 아이템을 보내는 마케터 (4,000~5,000자)

**하위 섹션:**
- 60분 회의, 녹음 파일을 올렸더니 5분 만에 회의록이 나왔다
- "회의에서 결정된 사항만 정리해줘", "각 담당자별 액션 아이템을 표로 만들어줘"
- 마케터가 사용한 핵심 기법: 오디오 소스 업로드 → 맞춤 질문 → 액션 아이템 추출
- 회의 문화가 바뀐 이야기: "회의록 쓰는 사람"이 사라진 팀
- [그림: 오디오 파일 업로드 → AI 분석 → 액션 아이템 표 워크플로우]

**핵심 정리:** 회의의 가치는 대화에 있다. 기록은 AI에게 맡기면 된다.

### 제4장: 100개의 설문에서 패턴을 찾아낸 HR 리더 (4,000~5,000자)

**하위 섹션:**
- 직원 만족도 설문 100건, 엑셀을 열기도 전에 한숨이 나왔다
- 설문 결과를 PDF로 변환하여 소스로 업로드
- "가장 자주 언급된 불만 사항 상위 5개를 뽑아줘", "긍정적 피드백과 부정적 피드백을 나눠서 정리해줘"
- HR 리더가 사용한 핵심 기법: 대량 텍스트 분석 → 패턴 추출 → 인사이트 보고서 생성
- 설문 분석에 이틀 걸리던 일이 30분으로 줄어든 이야기
- [그림: 설문 데이터에서 인사이트로 변환되는 과정 다이어그램]

**핵심 정리:** 양이 많을수록 AI가 더 빛난다. 사람은 100개를 읽으면 지치지만, AI는 지치지 않는다.

### 제5장: 노트북LM이란 무엇인가, '내 자료만 읽는 AI'라는 결정적 차이 (6,000~7,000자)

**하위 섹션:**

#### ChatGPT, Perplexity와 무엇이 다른가
- 비유: ChatGPT는 모든 책을 읽은 박학다식한 친구, Perplexity는 실시간 뉴스를 전해주는 기자, 노트북LM은 내 서류함만 뒤져주는 비서
- 핵심 차이: 소스 기반(Source-Grounded) 설계의 의미
- "내 자료만 읽기" 때문에 환각(Hallucination)이 근본적으로 줄어드는 이유
- 프라이버시: 내 데이터가 모델 학습에 사용되지 않는다는 보장
- [그림: 세 도구의 차이를 한눈에 보여주는 비교표]

| 비교 항목 | NotebookLM | ChatGPT | Perplexity |
|-----------|------------|---------|------------|
| 데이터 소스 | 내가 올린 자료만 | 인터넷 전체 | 실시간 웹 검색 |
| 핵심 강점 | 내 자료 분석 | 범용 생성 | 최신 정보 검색 |
| 환각 위험 | 매우 낮음 | 보통 | 낮음 |
| 프라이버시 | 높음 | 옵트아웃 필요 | 검색 기록 저장 |

#### 무료로 시작해도 됩니다, 플랜별 차이 한눈에 보기
- Free, Plus, Pro, Ultra 플랜별 핵심 차이를 표로 정리
- "무료로도 충분하다, 하지만 이런 경우에는 유료가 필요하다"는 가이드
- 학생 할인, 팀 도입 시 참고할 가격 정보
- [그림: 플랜별 기능 비교표]

**핵심 정리:** 노트북LM은 "내 자료만 읽는 AI"이다. 이 한 문장이 다른 모든 AI와의 차이를 설명한다.

## 제3부: 분석과 검증의 기술

**부제:** "여기서부터 결과가 달라집니다"

> 3부의 전략: 기본 사용법을 익힌 독자가 "중급"으로 올라가는 구간이다. 같은 도구를 쓰더라도 분석의 깊이가 달라지는 기법들을 다룬다.

### [활용법 01] 소스 비교 분석의 기술, 2개 비교부터 5개 이상 교차 분석까지 (3,000~4,000자)

**하위 섹션:**
- 2개 소스 비교: "이 두 보고서의 핵심 주장이 어디서 일치하고 어디서 갈리는지 정리해줘"
- 3~5개 소스 교차 분석: 여러 보고서에서 공통으로 나타나는 패턴 찾기
- 비교 분석 프롬프트 패턴 3가지
- 결과를 표로 정리하는 팁
- [그림: 2개 소스 비교 → 5개 소스 교차 분석 결과 예시]

**핵심 정리:** 비교 분석에서 가장 중요한 것은 "무엇을 비교할 것인가"를 먼저 정하는 것이다.

### [활용법 02] 노트를 소스로 재활용해서 2차 분석하기 (3,000~4,000자)

**하위 섹션:**
- 1차 분석 결과(노트)를 새 노트북의 소스로 업로드하는 과정
- 왜 2차 분석이 필요한가: 요약의 요약, 인사이트의 구조화
- 실전 예시: 경쟁사 분석 노트 3개 → 전략 종합 보고서
- [그림: 1차 분석 → 노트 저장 → 2차 노트북에 소스로 재활용 플로우차트]

**핵심 정리:** 노트북LM의 진짜 위력은 한 번의 분석이 아니라, 분석을 쌓아가는 것에 있다.

### [활용법 03] 맞춤 지시로 노트북의 성격 세팅하기 (3,000~4,000자)

**하위 섹션:**
- 맞춤 지시(Custom Instructions)란 무엇인가: 최대 10,000자의 지침 설정
- 노트북 수준 페르소나 vs 일회성 채팅 지시의 차이
- 실전 예시 3가지: "너는 마케팅 전략 컨설턴트다", "초등학생도 이해하게 설명해라", "표와 글머리 기호로 정리해라"
- 같은 소스, 다른 맞춤 지시에서 나오는 전혀 다른 결과 비교
- [그림: 동일 소스에 맞춤 지시 A vs B를 적용한 결과 비교]

**핵심 정리:** 같은 재료도 시즌에 따라 맛이 달라진다. 맞춤 지시는 노트북LM의 시즌이다.

### [활용법 04] AI가 틀릴 때 알아채는 법, 환각 잡고 소스 기반 팩트체크 (3,000~4,000자)

**하위 섹션:**
- 환각(Hallucination)이란 무엇인가: AI가 그럴듯하게 지어내는 현상
- 노트북LM은 왜 환각이 적은가: 소스 기반 설계의 원리
- 그래도 틀릴 수 있다: 출처 번호 클릭으로 원문 확인하는 습관
- 환각을 잡는 3단계: ① 출처 확인 ② 소스 원문과 대조 ③ 핵심 숫자와 날짜 재검증
- "이 정보의 출처를 보여줘"라는 질문을 습관화하기
- [그림: 답변 → 출처 번호 클릭 → 원문 확인 스크린샷]

**핵심 정리:** 노트북LM이 환각을 줄여주지만, 최종 검증은 사람의 몫이다. 출처를 확인하는 습관이 당신의 신뢰를 지킨다.

### [활용법 05] 소스 밖까지 확장하기, 딥 리서치로 웹까지 넓히는 법 (3,000~4,000자)

**하위 섹션:**
- 딥 리서치(Deep Research)란 무엇인가: 사용자의 질문 → 리서치 계획 수립 → 수백 개 웹사이트 탐색 → 상세 보고서 생성
- Fast Research vs Deep Research의 차이
- 딥 리서치가 최대 50개 소스를 자동 수집하는 과정
- 보고서와 소스를 노트북에 직접 추가하여 후속 분석에 활용하기
- 실전 예시: 시장 트렌드 리서치, 경쟁사 최신 동향 파악
- [그림: 딥 리서치 실행 → 리서치 계획 → 웹 탐색 → 보고서 생성 플로우]

**핵심 정리:** 노트북LM의 딥 리서치는 내 자료 안에 갇히지 않고 웹으로 나갈 수 있는 유일한 통로다.

### 3부를 마치며 (500~800자)
분석이 깊어졌다면, 이제 그 결과를 나 혼자 소화할 것인가, 남에게 보여줄 것인가. 4부와 5부가 그 두 갈래를 각각 다룬다.

## 제4부: 혼자 소화하는 기술

**부제:** "이동 중에도 일이 됩니다"

> 4부의 전략: 분석 결과를 나 혼자 소화하는 방법이다. 출퇴근길, 이동 중에도 활용 가능한 콘텐츠 변환 기법을 다룬다.

### [활용법 06] 내 자료를 팟캐스트와 영상으로 변환하기 (5,000~6,000자)

**하위 섹션:**

#### 오디오 오버뷰, 출퇴근길 업무 브리핑을 팀에 돌리는 법
- Audio Overview의 4가지 형식: Deep Dive(6-15분), Brief(1-2분) 등
- 대화형 Audio Overview: "Join"으로 AI 호스트와 직접 대화
- 80개 이상 언어 지원, 한국어 음성 가능
- 팀에 오디오 브리핑을 공유하는 실전 워크플로우

#### Video Overview, 텍스트가 설명 영상이 되는 법
- Video Overview의 생성 과정과 활용 사례
- 팀 교육 자료, 프로젝트 설명 영상으로 활용

#### Cinematic Video, 2026년 3월 신기능, 이렇게 씁니다
- Cinematic Video Overviews의 특징: Gemini 3 + Nano Banana Pro + Veo 3
- 기존 Video Overview와의 차이: 내레이션 슬라이드 → 완전 애니메이션
- Google AI Ultra($249.99/월) 전용 기능
- 어떤 상황에서 Cinematic Video가 빛나는가

#### 커스텀 지시로 같은 소스에서 전혀 다른 결과 만들기
- 동일 소스에 다른 커스텀 지시를 적용한 Audio/Video 결과 비교
- 톤, 대상 청중, 주제 설정 예시

- [그림: Audio Overview → Video Overview → Cinematic Video 비교표]

**핵심 정리:** 읽을 시간이 없으면, 들어라. 들을 시간도 없으면, 보라. 노트북LM은 정보의 형태를 바꿔준다.

### [활용법 07] 마인드맵으로 소스 구조 한눈에 보기 (3,000~3,500자)

**하위 섹션:**

#### 복잡한 보고서를 한 장의 지도로 만드는 법
- 마인드맵 생성 과정과 인터페이스
- 어떤 소스가 마인드맵에 적합한가

#### 마인드맵을 소스로 재활용하는 2차 분석
- 마인드맵을 캡처/저장하여 다른 노트북에 활용
- 마인드맵 기반 질문으로 세부 주제 파고들기

- [그림: 보고서 소스 → 마인드맵 변환 예시]

**핵심 정리:** 마인드맵은 숲을 본 다음 나무를 보게 해주는 도구다.

### [활용법 08] 플래시카드와 퀴즈로 내 머릿속에 새기기 (3,000~3,500자)

**하위 섹션:**

#### 읽고 잊어버리지 않는 법
- 플래시카드 자동 생성과 Got it/Missed it 기능
- 퀴즈로 핵심 내용 점검하기
- 진행 상황 저장 및 세션 간 이어서 학습

#### 5분 만에 팀 학습 자료로 전환하기
- 소스 업로드 → 플래시카드/퀴즈 생성 → 팀 공유
- 신입 교육, 프로젝트 킥오프에 활용하는 실전 예시

- [그림: 플래시카드 및 퀴즈 화면 예시]

**핵심 정리:** 플래시카드와 퀴즈는 나를 위한 도구이자, 팀을 위한 교육 도구이기도 하다.

### 4부를 마치며 (500~800자)
혼자 소화했다면, 이제 보여줄 차례다. 5부는 한 장으로 상대를 설득하는 기술이다.

## 제5부: 남에게 보여주는 기술

**부제:** "한 장으로 설득합니다"

> 5부의 전략: 분석 결과를 상대에게 보여주는 산출물을 만드는 기법이다. 슬라이드, 인포그래픽, 데이터표를 통해 설득력 있는 결과물을 빠르게 생성하는 방법을 다룬다.

### [활용법 09] 슬라이드 한 장으로 상대를 설득하는 법 (5,000~6,000자)

**하위 섹션:**

#### 맥킨지 스타일 슬라이드 자동 생성, 헤드라인과 서브타이틀과 출처 정렬 프롬프트
- 맥킨지 스타일 슬라이드의 핵심 원칙:
  - One Slide, One Message: 슬라이드 한 장에 메시지 하나
  - 헤드라인이 핵심 주장을 담는다
  - 헤드라인만 모아 읽으면 전체 스토리라인이 완성된다
- 일관성의 원칙:
  - 헤드라인 위치, 폰트 크기, 볼드 여부 통일
  - 서브타이틀 위치 고정
  - 본문 텍스트 시작점, 폰트 크기, 줄간격 통일
  - 차트, 표, 다이어그램 크기와 위치 일정하게 배치
  - 페이지 번호, 출처 표기, 로고 위치 고정
- 슬라이드 레이아웃 기본 구조 (맥킨지 표준):
  - 상단 15%: 헤드라인
  - 중앙 70%: 핵심 콘텐츠(차트, 표, 텍스트)
  - 하단 15%: 출처, 페이지 번호

#### 3단계 레이어링 프롬프트, 구조에서 상세, 상세에서 시각화
- 1단계: "이 소스의 핵심 인사이트를 슬라이드별 헤드라인으로 정리해줘"
- 2단계: "각 헤드라인을 뒷받침하는 근거와 데이터를 추출해줘"
- 3단계: "이 내용을 슬라이드 덱으로 만들어줘"

#### 노트북LM을 활용하는 워크플로우
- 소스 업로드 → 핵심 인사이트 추출
- 인사이트를 헤드라인으로 전환
- "헤드라인 테스트": 헤드라인만 모아 논리적 흐름 검증
- 데이터와 근거 추출
- 레이아웃 구조에 맞추어 슬라이드 배치

#### PPTX로 내보내고 편집하는 실전 워크플로우
- PPTX 내보내기 과정
- PowerPoint에서 미세 조정하는 팁

#### 슬라이드가 마음에 안 들 때 수정하는 법
- 개별 슬라이드 편집 기능 (2026년 2월 업데이트)
- 피드백을 제출하여 특정 슬라이드만 재생성
- [그림: 맥킨지 스타일 슬라이드 레이아웃 예시]

**핵심 정리:** 좋은 슬라이드는 예쁜 슬라이드가 아니라, 헤드라인만 읽어도 전체 이야기가 보이는 슬라이드다.

### [활용법 10] 인포그래픽으로 핵심 데이터 한 장으로 압축하기 (3,000~3,500자)

**하위 섹션:**

#### 의사결정권자를 위한 원페이저 만들기
- 인포그래픽의 10종 스타일과 적합한 사용 상황
- 가로(16:9) vs 세로(9:16) 형식 선택 가이드
- 대상 청중별 스타일 추천

#### 숫자를 그림으로 바꾸는 프롬프트 패턴
- 데이터 중심 인포그래픽 생성 프롬프트
- "이 데이터의 핵심 수치를 인포그래픽으로 만들어줘"
- [그림: 인포그래픽 스타일 10종 미리보기]

**핵심 정리:** 임원에게 20페이지 보고서를 주지 마라. 한 장의 인포그래픽이 20페이지보다 강하다.

### [활용법 11] 데이터표로 소스 내용 구조화하기 (3,000~3,500자)

**하위 섹션:**

#### 비교표, 요약표, 타임라인 한 번에 만들기
- "이 세 보고서의 핵심 내용을 비교표로 정리해줘"
- 타임라인 표, 장단점 비교표, SWOT 분석표 생성 프롬프트
- Google Sheets로 내보내기

#### 표를 보고서 초안으로 연결하는 법
- 생성된 표를 노트에 저장 → 보고서 초안의 골격으로 활용
- [그림: 소스 → 데이터표 → 보고서 초안 변환 플로우]

**핵심 정리:** 표는 생각을 정리하는 가장 빠른 형식이다. 정리된 생각은 보고서가 된다.

### 5부를 마치며 (500~800자)
산출물이 달라졌다. 6부는 이 도구를 실제 업무 현장에 착지시키는 시나리오다.

## 제6부: 월요일 아침부터 바로 쓰는 실전 시나리오

**부제:** "이 챕터만 읽어도 오늘 업무가 달라집니다"

> 6부의 전략: 3~5부에서 익힌 기법을 실제 업무 상황에 적용하는 구간이다. 각 시나리오는 독립적으로 읽어도 바로 따라할 수 있게 구성한다.

### [활용법 12] 업무 보고서 30분 만에 초안 뽑기 (3,000~4,000자)

**하위 섹션:**

#### 맥락+상황+목적+출력 형식, 팀장을 위한 프롬프트 공식
- 프롬프트 구조: [맥락] + [상황] + [목적] + [출력 형식]
- 예시: "나는 마케팅 팀장이다. 이번 분기 캠페인 결과 보고서를 분석하고 있다. 임원 보고용 핵심 요약을 작성해줘. 글머리 기호 형식으로."

#### 바로 쓰는 프롬프트 패턴 5선
1. 주간 보고서 초안 프롬프트
2. 분기 실적 분석 프롬프트
3. 프로젝트 현황 보고 프롬프트
4. 시장 분석 보고서 프롬프트
5. 제안서 초안 프롬프트

- [그림: 프롬프트 공식 구조 다이어그램]

**핵심 정리:** 좋은 프롬프트 = 좋은 질문이다. 맥락, 상황, 목적, 형식을 넣으면 답이 달라진다.

### [활용법 13] 회의록에서 액션 아이템 자동 추출하기 (3,000~4,000자)

**하위 섹션:**
- 회의 녹음/녹취록을 소스로 올리는 과정
- "결정 사항만 정리해줘", "담당자별 액션 아이템을 표로 만들어줘"
- 회의록 → 액션 아이템 → 후속 메일 초안까지 한 번에
- 주간 회의, 프로젝트 킥오프, 임원 미팅별 프롬프트 패턴
- [그림: 회의 녹음 → 업로드 → 회의록 + 액션 아이템 추출 워크플로우]

**핵심 정리:** 회의에서 가장 중요한 것은 "누가 무엇을 언제까지 하는가"다. 나머지는 AI가 정리한다.

### [활용법 14] 경쟁사 분석 보고서 만들기, 교차 비교에서 전략 인사이트까지 (3,000~4,000자)

**하위 섹션:**
- 경쟁사 IR 보고서, 뉴스, 블로그를 소스로 수집
- 교차 비교 분석 프롬프트: "이 세 경쟁사의 핵심 전략, 강점, 약점을 비교표로 정리해줘"
- 전략 인사이트 도출: "이 비교 결과를 바탕으로 우리 팀이 주목해야 할 기회와 위협을 정리해줘"
- 딥 리서치로 최신 동향까지 추가
- [그림: 경쟁사 소스 3개 → 교차 비교표 → 전략 인사이트 보고서]

**핵심 정리:** 경쟁사 분석의 핵심은 "비교"다. 노트북LM은 비교를 가장 잘한다.

### [활용법 15] 고객 피드백에서 숨은 패턴 찾기, VOC와 설문과 리뷰 분석 (3,000~4,000자)

**하위 섹션:**
- VOC, 설문 결과, 리뷰 데이터를 PDF/텍스트로 변환하여 소스 업로드
- "가장 많이 언급된 불만 상위 5개", "긍정 피드백 vs 부정 피드백 분류"
- 패턴 분석: "시간 순으로 불만이 어떻게 변화했는지 정리해줘"
- 인사이트를 인포그래픽이나 보고서로 변환
- [그림: 설문/리뷰 데이터 → 패턴 분석 → 인사이트 보고서]

**핵심 정리:** 100개의 피드백에서 패턴을 찾는 것은 사람에게는 이틀, AI에게는 5분이다.

### [활용법 16] 신입 온보딩 지식베이스 만들기, 사내 문서로 스스로 학습하는 노트북 구축 (3,000~4,000자)

**하위 섹션:**
- 사내 규정, 업무 매뉴얼, FAQ를 소스로 업로드
- 신입이 직접 질문할 수 있는 "팀 AI 비서" 구축
- 플래시카드와 퀴즈로 온보딩 학습 자료 자동 생성
- 노트북 공유 설정으로 팀 전체가 접근하도록 구성
- [그림: 온보딩 지식베이스 구축 워크플로우]

**핵심 정리:** 신입이 물어볼 때마다 시간을 쓰지 마라. 노트북LM에게 한 번 가르치면, 신입이 100번 물어도 된다.

### 6부를 마치며 (500~800자)
개인의 생산성은 바뀌었다. 마지막 질문이 남았다. 이것을 나 혼자만 쓸 것인가, 아니면 팀의 무기로 만들 것인가.

## 제7부: 나에서 팀으로, 도구에서 시스템으로

**부제:** "혼자 잘하는 것과 함께 잘하는 것은 다릅니다"

> 7부의 전략: 개인 도구에서 팀/조직의 시스템으로 확장하는 방법이다. 협업, 다른 AI 도구와의 역할 분담, 실제 루틴을 다룬다.

### [활용법 17] 나만의 워크플로우 설계하기, 반복 업무를 노트북LM 루틴으로 (3,000~4,000자)

**하위 섹션:**
- 반복 업무 식별: 매주/매월 반복되는 보고, 분석, 정리 작업
- 노트북 템플릿 만들기: 목적별 노트북 구조 설계
- 맞춤 지시를 활용한 일관된 출력 유지
- 워크플로우 자동화: 소스 업데이트 → 분석 → 산출물 생성의 루틴화
- [그림: 주간 업무 워크플로우 설계도]

**핵심 정리:** 좋은 루틴은 매번 처음부터 시작하지 않게 해준다. 노트북LM으로 반복을 자동화하라.

### [활용법 18] 팀과 조직으로 확장하기 (4,000~5,000자)

**하위 섹션:**

#### 노트북 공유와 실시간 협업 설정하는 법
- Viewer/Editor 권한 설정 방법
- 팀 노트북 운영 규칙 정하기
- Public 노트북 활용 시 주의사항

#### Google Docs로 내보내고 팀과 함께 편집하기
- Studio Panel에서 Google Docs/Sheets로 내보내기
- 내보낸 문서의 동기화 제한 이해하기
- 팀 공동 편집 워크플로우

#### 조직 도입 체크리스트, 처음 제안할 때 쓰는 말
- 팀장이 임원에게 노트북LM 도입을 제안하는 시나리오
- ROI 설명: 시간 절감, 분석 품질 향상, 지식 관리 효율화
- 보안/프라이버시 우려에 대한 답변 가이드
- 도입 체크리스트 10항목

- [그림: 팀 협업 워크플로우와 권한 설정 가이드]

**핵심 정리:** 도구를 혼자 쓰면 개인의 생산성이 바뀌고, 함께 쓰면 팀의 문화가 바뀐다.

### [활용법 19] 노트북LM이 안 되는 것, 다른 AI에게 넘기는 법 (4,000~5,000자)

**하위 섹션:**

#### ChatGPT와 노트북LM, 무엇이 다르고 언제 바꿔 쓰는가
- ChatGPT가 더 나은 상황: 창의적 글쓰기, 코드 생성, 아이디어 브레인스토밍
- 노트북LM이 더 나은 상황: 내 자료 분석, 소스 기반 팩트체크, 멀티미디어 변환

#### Perplexity, Claude와 역할 분담하는 3가지 패턴
- 패턴 1: Perplexity(수집) → NotebookLM(분석/정리)
- 패턴 2: ChatGPT(아이디어) → NotebookLM(구조화/검증)
- 패턴 3: Claude(심층 분석) → NotebookLM(산출물 생성/공유)

#### 도구를 바꾸는 판단 기준 한 장으로 정리
- 의사결정 플로우차트: "이 작업에 어떤 AI를 쓸 것인가?"
- [그림: AI 도구 선택 의사결정 플로우차트]

**핵심 정리:** 최고의 도구는 만능 도구가 아니라, 적재적소에 쓰는 도구다.

### [활용법 20] 팀장의 노트북LM 루틴, 월요일부터 금요일까지 (4,000~5,000자)

**하위 섹션:**

#### 실제 일주일 사용 패턴
| 요일 | 활동 | 사용 기능 |
|------|------|-----------|
| 월 | 주간 보고 자료 분석 | 소스 업로드, 비교 분석 |
| 화 | 회의록 정리, 액션 아이템 추출 | 오디오 소스, 표 생성 |
| 수 | 경쟁사/시장 동향 분석 | 딥 리서치, 인포그래픽 |
| 목 | 팀 보고서 초안 작성 | 프롬프트 패턴, 슬라이드 |
| 금 | 주간 리뷰, 다음 주 준비 | 마인드맵, 노트 정리 |

#### 팀에 처음 소개할 때 쓰는 30분 데모 스크립트
- 5분: 왜 노트북LM인가 (1부의 사례 요약)
- 10분: 함께 해보기 (소스 업로드 → 첫 질문 → 결과 확인)
- 10분: 우리 팀에 적용할 수 있는 3가지 시나리오
- 5분: Q&A

- [그림: 팀장의 일주일 노트북LM 루틴 타임라인]

**핵심 정리:** 루틴이 되면 도구는 사라진다. 남는 것은 결과뿐이다.

### 7부를 마치며 (500~800자)
임원은 처음에 팀장을 지켜봤다. 지금은 이사회 보고 전날 밤, 혼자 세 개 사업부 자료를 통합한다. 에필로그는 그 밤의 이야기다.

## 에필로그: 지금 노트북을 여십시오

**핵심 내용:**
- 1부의 네 사람 이야기를 간결하게 되새긴다:
  - 팀장은 오늘도 5분 만에 보고서를 정리했다
  - 기획자는 혼자 오전 안에 시장조사를 끝냈다

## 학습 흐름 설계

독자의 학습 경로는 다음과 같이 설계되었다:

```
프롤로그 (호기심)
  → 1부 (동기부여: "저 사람들은 뭘 쓰는 거지?")
    → 2부 (시작: "나도 해볼까?")
      → 3부 (심화: "같은 도구인데 결과가 다르다")
        → 4부 (개인 활용: "혼자 소화")
        → 5부 (공유 활용: "남에게 보여주기")
          → 6부 (실전: "월요일부터 바로")
            → 7부 (확장: "나에서 팀으로")
              → 에필로그 (마무리: "지금 열어라")
```

3부까지는 순서대로 읽는 것을 권장한다. 4부~7부는 필요에 따라 골라 읽을 수 있다.
